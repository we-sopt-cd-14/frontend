## Commit Convention

`종류: 내용` 포맷으로 메시지 작성

- **feat:** 새로운 기능 추가
- **fix:** 버그 수정
- **refactor:** 리팩터링
- **docs:** 설명 문서 변경
- **perf:** 성능 개선
- **test:** 테스트코드 추가
- **chore:** 프로젝트 설정 등 변경
